AI collapsed the struggle that builds real expertise.

New developers reach feature-shipping competence without earning the mental model of the codebase. When AI is wrong, they have nothing to fall back on.

Experienced developers spend their day reviewing AI output. Pattern-matching skills atrophy because they're skimming generated diffs instead of authoring carefully.

CodeDojo is deliberate practice that reverses both. Active recall for newcomers. Adversarial sharpening for experts.

Practice

Two modes, one engine.

Both modes run against real code — your code — with a Socratic AI coach that refuses to give answers.

Learn mode

Newcomer

CodeDojo reverts a real feature commit from your repo's history. You get a stripped feature description with implementation details removed. Your job: reimplement it until the original tests pass.

The coach offers Socratic hints — reflective questions, file pointers, conceptual nudges — but never the answer. Every hint costs score.

Who it's for
Devs new to a codebase
Ground truth
Original commit + tests
Session length
30 – 90 min
Win condition
Tests pass + sound approach
Review mode

Reviewer

CodeDojo injects one plausible AST-level mutation into a Go file — a flipped boundary, a dropped error check, a negated conditional. Tests may or may not catch it. That's part of the lesson.

You inspect the code, run tests, narrow it down, and submit the file, line range, and diagnosis. Partial credit for proximity.

Who it's for
Devs familiar with the codebase
Ground truth
Mutation engine log
Session length
5 – 20 min
Win condition
Localized + diagnosed

Internals

Architecture.

Two thin mode layers on a shared spine of five interfaces. Reviewer and newcomer share infra but keep domain logic separate.

01 — entry
cmd/codedojo CLI entrypoint (cobra)
↓
02 — cli
internal/cli cobra commands + REPLs
↓
03 — infrastructure
internal/repo clone / open / detect
internal/store sqlite / memory
internal/sandbox local / docker driver
internal/session state / scoring
↓
04 — domain
internal/modes reviewer + newcomer
↔
internal/coach mock / anthropic / ollama

sandbox.Driver

Starts isolated sessions. Exec, WriteFile, ReadFile, Diff. Docker preferred; local fallback.

coach.Coach

Hint() and Grade(). Output-validator wrapped. Retries with stricter prompt on code leaks.

session.Manager

State machine: Created → Running → Submitted → Graded → Closed. Hint budget + streaks.

store.Store

SQLite persistence for sessions, events, scores, mutation logs, history cache.

repo.Repo

go-git wrapper. Clone, OpenLocal, DetectLanguage. Never mutates the original path.

mutate.Engine

AST-level Go mutations. Boundary, conditional, error-drop, slice-bounds operators.

Security

Sandbox model.

CodeDojo runs build and test commands from repositories that may be unfamiliar or intentionally broken. Two driver modes share one interface.

Docker driver recommended

  • Container root filesystem read-only
  • --network=none by default
  • Only /workspace mounted read-write
  • CPU, memory, and PID limits enforced
  • All Linux capabilities dropped
  • no-new-privileges set
  • Container and tmpdir removed on close

Falls back to local with a warning when Docker is unreachable. Use Docker for any unfamiliar repository.

Local driver dev only

  • Copies repo into a tmpdir
  • Runs commands on the host directly
  • No network isolation
  • No CPU or memory limits
  • Commands inherit host environment
  • Not a security boundary

For development, tests, and trusted repos only. Switch to Docker before running unfamiliar code.

Get started

Quickstart.

Prerequisites: Go 1.23+, Git. Docker is optional but recommended for sandboxing.

Build & configure 
# clone and build
git clone https://github.com/dhruvmishra/codedojo
cd CodeDojo
make build

# first-run wizard: pick LLM backend, API key,
# and default difficulty
./bin/codedojo init

# optional: build Docker sandbox images
make images
Reviewer mode Go 
./bin/codedojo review \
  --repo testdata/sample-go-repo \
  --difficulty 2 --budget 3

# inside the REPL
tests                    # run tests
cat calculator.go        # inspect file
hint nudge               # ask coach
submit calc.go:13 \
  "boundary comparison"  # submit
Newcomer mode Go 
./bin/codedojo learn \
  --repo testdata/sample-go-repo \
  --difficulty 2 --budget 3

# inside the REPL
tests                    # see failures
cat calculator.go        # inspect code
write calculator.go      # edit file
diff                     # review changes
submit                   # grade it
Local web UI Browser 
# serve the local web app
./bin/codedojo serve \
  --repo testdata/sample-go-repo \
  --port 8080

# opens at http://localhost:8080
# complete sessions in the browser;
# no account or cloud service

./bin/codedojo status   # past sessions
./bin/codedojo stats    # aggregate stats
